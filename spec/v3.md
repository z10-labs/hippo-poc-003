# Forge v3 — Multi-Priority Queue Lanes

This spec extends the v2 implementation. All F-01 through F-08 features are already in place. Implement F-09 only.

## F-09 — Multi-Priority Queue Lanes

The v1 job queue (F-01) accepts a `priority` integer per job but currently uses a single fetch order — jobs are dispatched in insertion order. Replace this with true priority-lane execution: jobs with higher priority values are always dispatched before lower-priority jobs in the same queue, without starving low-priority work indefinitely.

**Requirements**

- Priority is an integer; higher value = higher urgency (default: 0)
- Within a queue, a high-priority eligible job must be dispatched before any lower-priority eligible job
- Low-priority jobs must not starve: a job that has been waiting longer than a configurable `maxWaitMs` threshold must be promoted and dispatched regardless of other pending higher-priority jobs
- Priority lanes must integrate with the existing concurrency enforcement (F-02) — the per-type concurrency limit still applies regardless of priority
- Priority must be reflected in `GET /jobs` results and `GET /queues` queue depth breakdowns (show depth per priority band: high / normal / low)
- Starvation promotion events must be written to the audit log (F-08) as a new event type `job.promoted`

**Constraints**

- No new tables unless strictly necessary — prefer extending the existing schema
- The solution must not regress enqueue latency (p99 < 10ms)
- Starvation prevention must survive restart: a job's wait time is measured from its original `created_at`, not from process start

**Decision surface**

Priority dispatch touches the worker polling query (which must now order by priority and age), the starvation threshold (where does it live — per-queue config, per-job-type registration, or global), and whether the promotion event reuses the audit log machinery from F-08 or writes through a separate path. Each of these is a separate decision that depends on prior choices. Query the index before logging each one.
